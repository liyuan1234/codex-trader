# Dual Momentum Strategy Walkthrough

## What the code does

Source: [`quant_trader/strategies.py`](/Users/liyuan/Desktop/codex-image-agent/quant-trading-analyzer/quant_trader/strategies.py)

For each ticker:

1. Compute a 50-day moving average.
2. Compute a 200-day moving average.
3. Set the raw signal to `1` when `fast_ma > slow_ma`, otherwise `0`.
4. Estimate 20-day annualized volatility.
5. Scale the position by `volatility_target / realized_volatility`.
6. Cap leverage at `1.0`.
7. Cross-sectionally normalize all active positions so total gross exposure is `1.0`.

This is a long-only trend model. It buys assets already in an uptrend and allocates less capital to the more volatile names.

## Buy condition

- Buy or hold long when `50-day MA > 200-day MA`.
- Size is larger when realized 20-day volatility is lower.

## Sell condition

- Exit a ticker when `50-day MA <= 200-day MA`.
- The code does not use a separate profit target.

## Stop loss

- There is no explicit stop-loss order in the current implementation.
- Risk reduction happens indirectly through:
  - trend filter turning off when fast MA falls below slow MA
  - volatility scaling reducing weight in noisier names
  - cross-sectional normalization across the whole basket

## What an order would look like

With this strategy:

- a ticker crossing above the 200-day trend filter leads to a `BUY`
- a ticker crossing below the filter leads to a `SELL`
- otherwise the system may rebalance position size without changing the trend direction

## Limitations

- No hard stop-loss
- No transaction scheduling beyond daily rebalance assumptions
- No options logic in the current implementation
