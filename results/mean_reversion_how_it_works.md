# Mean Reversion Strategy Walkthrough

## What the code does

Source: [`quant_trader/strategies.py`](/Users/liyuan/Desktop/codex-image-agent/quant-trading-analyzer/quant_trader/strategies.py)

For each ticker:

1. Compute a 20-day rolling mean.
2. Compute a 20-day rolling standard deviation.
3. Compute z-score: `(price - rolling_mean) / rolling_std`.
4. Set raw signal:
   - `+1` if `zscore < -1.5`
   - `-1` if `zscore > 1.5`
   - `0` if `|zscore| < 0.5`
5. Cross-sectionally normalize active positions so total gross exposure is `1.0`.

This is a short-horizon contrarian strategy. It buys sharp underperformance and shorts sharp overperformance, expecting price to revert toward the recent mean.

## Buy condition

- Buy when the z-score drops below `-1.5`.
- That means price is materially below the 20-day average.

## Sell condition

- Close the position when `|zscore| < 0.5`.
- Short when the z-score rises above `+1.5`.

## Stop loss

- There is no explicit stop-loss order in the current implementation.
- The only coded exit is the z-score reverting inside the `0.5` neutral band.
- This is one reason the backtest was weak: reversions can keep going against the position before mean reversion happens.

## What an order would look like

With this strategy:

- `BUY` when z-score falls below `-1.5`
- `SELL` when a long position reverts back inside `|z| < 0.5`
- `SELL SHORT` when z-score rises above `+1.5`
- `BUY TO COVER` when a short position reverts back inside `|z| < 0.5`

## Limitations

- No hard stop-loss
- No regime filter to disable the strategy during strong trends
- No borrow, financing, or short-locate modeling
- No options logic in the current implementation
